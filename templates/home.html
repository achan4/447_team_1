Visualizing Crime In Baltimore City

CMSC 447 Spring 2022 Team 1

Data Source

The data used for this website comes from information available to the public here. At this site, there is a CSV file available for download, which is what this project uses as a source for data.

The CSV file used containing data of Baltimore City crime is over 500,000 entries long. The data is organized into different sections that we can use to graph and observe trends. The sections used to create this web application are the time and date, description, whether the crime was inside or outside, the weapon used, the district, and the neighborhood.

Database

For this project we decided to use MongoDB to hold the data from our data source. The database contains many different collections that stores the counts of crimes sorted by different categories.

We store the various collections for later use in our map and graphs. The main objective of this is to speed up the time it takes for the database to send information back to the user in order to generate a more responsive user experience.

Development Tools

Our team has chosen to use several tools to facilitate the development process. Almost all of the tools were new for us so it was a great learning experience.

This web application uses the Flask micro web framework for our backend development. For our front end development, our team has used HTML templates, CSS, and JavaScript. For our map of the city of Baltimore, we use Leaflet, which is an open source JavaScript library to build wab mapping applications.
